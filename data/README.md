### Data construction

The training data was built in layers, each addressing a specific gap:

1. **LLM paraphrases** (3,831) â€” Generated by prompting an LLM with tool descriptions and type mappings. Provides naturalistic diversity but only covers direct graph edges.
2. **Deterministic templates** (2,018) â€” Template-based generation from the graph snapshot. Covers all direct (source, target) pairs with structured query patterns.
3. **2-hop examples** (1,489) â€” Generated for all 251 (source, target) pairs reachable via 2-hop paths but not directly connected. Without these, the model had zero training examples for multihop test queries.
4. **Anti-self-referential** (280) â€” Vague queries ("Check the credentials", "Is anything running?") labeled as Platformâ†’X. Prevents the model from predicting Xâ†’X for underspecified queries.
5. **Hard negatives + synonyms** (760) â€” Domain-specific confusing pairs and informal terminology mappings.
6. **Targeted** (333) â€” Specific examples for persistent error patterns identified during iterative training.

### Key design decisions

- **Self-loop downsampling**: Xâ†’X examples (e.g., Credentialâ†’Credential) were downsampled to 30% for types where Platformâ†’X also exists. Without this, the model predicted self-referential types for vague queries (e.g., "list credentials" â†’ Credentialâ†’Credential instead of Platformâ†’Credential).
- **Sqrt-scaled class weights** during training correct for remaining source/target imbalance.
- **Zero test/train overlap**: All 47 test queries are filtered out during data construction.
